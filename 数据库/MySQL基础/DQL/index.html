# 1. 基础查询

# 1.1 语法

SELECT
	查询列表 
FROM
	表名；

查询列表可以是：表中的字段、常量值、表达式、函数。

  • 查询表中的字段(查询所有字段使用*)

    SELECT
	last_name,
	email 
FROM
	employees;

# 1.2 起别名

使用AS关键字或者使用空格

  • SELECT
	last_name AS n,
	email AS e 
FROM
	employees;

  • SELECT
	last_name n,
	email e 
FROM
	employees;

  • SELECT
	last_name AS n,
	email AS e 
FROM
	employees em;

# 1.3 去重

使用DISTINCT关键字

SELECT DISTINCT
	department_id 
FROM
	employees;

# 1.4 使用concat实现连接

使用concat函数可以实现连接成一个字段

SELECT
	CONCAT( last_name, first_name ) AS `name` 
FROM
	employees;

# 2. 条件查询

# 2.1 语法

SELECT
	查询列表 
FROM
	表名 
WHERE
	查询条件

# 2.2 分类

  • 按条件表达式查询： >、<、=、!= 、>=、<=
  • 按逻辑表达式查询：and、or、not，逻辑运算符的作用就是连接条件表达式。
  • 模糊查询：属于条件表达式查询的特殊，like、between and、is null、in

使用like关键字进行模糊查询时，_用来匹配一个字符，%用来匹配多个字符。

# 3. 排序查询

# 3.1 语法

SELECT
	查询列表 
FROM
	表名
[WHERE 查询条件]
ORDER BY
	排序列表 DESC / ASC

DESC为降序；ASC为升序，默认为升序，可以不写。

# 4. 常见函数

概念：类似于java的方法，将一组逻辑预警封装在方法体中，对外暴露方法名

优点：隐藏了实现细节，提高重用性

语法：

SELECT
	函数名(实参列表)
FROM
	表名
[WHERE 查询条件]

# 4.1 分类

  • 单行函数：如concat、length、ifnull等
  • 分组函数：功能是做统计使用，又称为统计函数、聚合函数、组函数

# 4.2 单行函数

单行函数可以分为：

  • 字符函数
    • length(str)：用来获取字符串字节长度。
    • concat(str1, str2)：拼接字符
    • upper(str)、lower(str)
  • 数学函数
  • 日期函数
    • now()；返回当前系统日期+时间
    • curdate()：返回当前日期不包含时间
    • curtime()：返回当前时间不包含日期
  • 流程控制函数
  • 其他函数

# 4.2 分组函数

分组函数包括：sum求和、avg平均值、max最大值、min最小值、count计算个数

# 5. 分组查询

# 5.1 语法

SELECT
	分组函数
FROM
	表名
[WHERE 查询条件]
GROUP BY 分组条件
HAVING 分组后筛选条件

注意：查询列表必须特殊，要求是分组函数和group by后出现的字段，where查询条件是跟在from后，而having是分组后的筛选

# 6. 连接查询

当查询结果在多个表中就需要使用到连接查询。

# 6.1 分类

按年代分类：

  • sql92标准：在mysql中仅仅支持内连接

  • sql99标准【推荐】：在mysql中支持内连接+外连（左外和右外）+交叉连接

    语法：

    SELECT
	查询列表 
FROM
	表1 别名 【	INNER JOIN /LEFT JOIN /RIGHT JOIN】
  表2 别名 ON 查询条件 
【WHERE 筛选条件】
【GROUP BY 分组】
【HAVING 分组后筛选条件】
【ORDER BY 排序】

按功能分类：

  • 内连接

    • 等值连接：

      • sql92标准
      SELECT
	girl.`name` AS girl,
	boy.boyName AS boy 
FROM
	beauty girl,
	boys boy 
WHERE
	girl.boyfriend_id = boy.id

    • 非等值连接：查询条件不是=

    • 自连接：

      • sql92标准
      SELECT
	e1.employee_id AS '员工id',
	e1.last_name AS '员工',
	e2.last_name AS '上级' 
FROM
	employees e1,
	employees e2 
WHERE
	e1.manager_id = e2.employee_id1

      • sql99标准

        SELECT
	e2.employee_id AS '员工id',
	e2.last_name AS '员工',
	e1.last_name AS '上级' 
FROM
	employees e1
	INNER JOIN employees e2 ON e1.employee_id = e2.manager_id

  • 外连接

    应用场景：用于查询一个表中有，而另外一个表中没有的记录。外连接的查询结果为主表中的所有记录，如果从表中有和它匹配的，则显示匹配的值，否则显示null

    • 左外连接：LEFT JOIN左边的表是主表
    • 右外连接：RIGHT JOIN右边的表是主表
    • 全外连接

  • 交叉连接

# 7. 子查询

概念：出现在其他语句内部的select语句，称为子查询或内查询，内部嵌套其他select语句的查询，被称为主查询或外查询。

示例：

SELECT
	first_name 
FROM
	employees 
WHERE
	department_id IN (
	SELECT
		department_id 
	FROM
		departments 
	WHERE
	location_id = 1700 
	)

# 8. 分页查询

语法：

SELECT 查询列表
FROM 表
【join type JOIN 表2
  ON 连接条件
  WHERE 筛选条件
  GROUP BY 分组字段
  HAVING 分组后筛选
  ORDER BY 排序字段
】
LIMIT offset,size;

offset为要显示记录的起始索引（起始索引从0开始），size为要显示的记录条数

在使用时可以使用 LIMIT (page - 1) * size, size，例如查询第一页的10条数据：limit 0，10

# 9. union联合查询

union 联合 合并：将多条查询语句的结果合并成一个结果
