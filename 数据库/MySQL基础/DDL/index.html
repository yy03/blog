DDL语言

2020-11-19
数据库
MySQL

# 1. 介绍

DDL：数据定义语言。可以分为对库和表的创建、修改、删除管理。

  • 创建：create
  • 修改：alter
  • 删除：drop

# 2. 库的管理

# 2.1 库的创建

基本语法：

create database [if not exists] 库名：

# 2.2 库的修改

一般不修改

# 2.3 库的删除

基本语法：

drop database [if exists] 库名;

# 3. 表的管理

# 3.1 表的创建

语法：

create table [if not exists] 表名 (
    字段名 字段的类型(长度) [约束]，
    字段名 字段的类型(长度) [约束],
    字段名 字段的类型(长度) [约束]，
    ...
    字段名 字段的类型(长度) [约束]
)

# 3.2 表的修改

  • 修改列名：

    alter table 表名 change column 旧列名 新列名 类型;

  • 修改列的类型或约束

    alter table 表名 modify column 列名 类型或约束;

  • 添加新列：

    alter table 表名 add column 列名 类型;

  • 删除列：

    alter table 表名 drop column 列名;

  • 修改表名

    alter table 表名 rename to 新表名;

# 3.3 表的删除

drop table [if exists] 表名;

# 4. 常见约束

六大约束：

  • NOT NULL：非空约束
  • DEFAULT：默认约束，用于保证字段名有默认值
  • PRIMARY KEY：主键约束，用于保证字段名的唯一性并且非空
  • UNIQUE：唯一约束，用于保证该字段名的唯一性，但是允许为空
  • CHECK：检查约束，MySQL中不支持
  • FOREIGN KEY：外键约束，用于限制两个表的关系，在从表中添加外键约束，用于引用主表中某列的值。

约束的添加分类：

  • 列级约束：六大约束语法上都支持，但是外键约束没有效果
  • 表级约束：除了非空和默认，其他都支持；添加表级外键约束语法为：CONSTRAINT 约束名 FOREIGN KEY (从表字段) REFERENCES 主表(主表字段)
create table 表名(
    字段名 类型 列级约束,
    字段名 类型,
    表级约束
)
